Properties of RBT


Red-Black tree is a self-balancing Binary Search Tree(BST) where every node follows the following rules:
Rule  1:      Every node has a color either red or black.
Rule 2:      Root of tree is always black.
Rule 3:      There are no two adjacent red nodes (A red node cannot have a red parent or red child).
Rule 4:      Every path from root to a NULL node has same number of black nodes.





Key Insertion Operation


If T is a non-empty tree, then we do the following:
 1:     Use the BST insert algorithm to add K to the tree.
2:     Color the node containing K red.
3:     Restore red-black tree properties (if necessary).



To restore the violated property, we use:
 1:     Recoloring
2:     Rotation (Left, Right, Double)



The algorithm has mainly two cases depending upon the color of Uncle(Sibling of Parent node)
 1:     If Uncle is red, we do recoloring.
2:     If Uncle is black, we do rotations and/or recoloring.



Let X be the newly inserted node:
 1:     Change the color of Parent and Uncle as BLACK
2:     Change the color of Grand-Parent as RED
3:     Make X's Grand-Parent as the new X.
4:     Repeat the same above steps on the newly obtained X.



If X's Uncle is BLACK and X's parent is not BLACK then there can be four configurations for X:



➊   Left - Left Case


Right Rotation


➋   Left - Right Case


Left - Right Rotation


➌   Right - Right Case


Left Rotation


➍   Right - Left Case


Right - Left Rotation


Deletion of a node


On deleting a node, the number of black nodes in a path changes.

Two things to keep in mind:
 1:     If a black node is deleted and replaced by a black child, then the child is double black.
2:     Also we check color of sibling of delete node.



CASE - 1: Sibling is black and both of it's children are black
 1:     Here we need to see the color of Parent node.
2:     If it is RED, then change it to Black.
                               ( OR )
2:     If it was initially BLACK, then change it to Double Black.
3:     Also change the color of sibling ot RED.



CASE - 2: Sibling is BLACK and it's Left-Child is RED
 1:     Change the color of Sibling to RED and it's Child to BLACK.
2:     Now do Right Rotation at the Sibling.
3:     The Case is now similar to Case - 3 which follows next.



CASE - 3: Sibling is BLACK and it's Right-Child is RED
 1:     Change the color of Right-Child to BLACK.
2:     Exchange the colors of Parent and Sibling.
3:     Do Left Rotation at the Parent.
4:     Remove Double Black.



CASE - 4: Sibling color is RED
 1:     Exchange the colors of Parent and Sibling.
2:     Do Left Rotation at the Parent.
3:     Now this Case has been converted to Case - 1.
